# 🍽️ Система управления кафе

Веб-приложение для автоматизации работы небольшого кафе или ресторана. Система разделена на три функциональные зоны: **касса (зал)**, **кухня** и **статистика**. Проект позволяет официантам быстро принимать заказы, поварам — видеть актуальные задачи, а управляющим — отслеживать финансовые показатели. В основе лежит база данных SQLite, ORM SQLAlchemy для работы с данными и NiceGUI для создания современного веб-интерфейса.

### Возможности системы

**Для зала (официантов):**
- Просмотр меню с разделением по категориям
- Формирование заказа: добавление блюд в корзину, изменение количества
- Отправка заказа на кухню с указанием номера стола
- Просмотр всех активных заказов и их статусов
- Печать (просмотр) кассового чека
- Закрытие счета (удаление выполненного заказа)

**Для кухни (поваров):**
- Отображение текущих заказов в порядке очереди
- Визуальное выделение заказов, которые готовятся дольше 15 минут
- Смена статуса: «в обработке» → «готовится» → «готов»
- Автоматическое обновление экрана каждые 10 секунд

**Для аналитики (управляющего):**
- Отдельная страница с подробной статистикой
- Ключевые метрики: выручка за сегодня, количество заказов в работе, средний чек
- График динамики выручки за последние 7 дней
- Топ-5 популярных блюд за последние 30 дней с визуализацией
- Общая статистика за всё время

### Структура базы данных

**Category** — категории блюд
- `id` — идентификатор
- `name` — название категории (уникальное)

**Dish** — блюда
- `id` — идентификатор
- `name` — название блюда
- `price` — цена
- `category_id` — ссылка на категорию

**Order** — заказы
- `id` — идентификатор
- `table_number` — номер стола
- `total` — общая сумма заказа
- `status` — статус (в обработке / готовится / готов)
- `created_at` — дата и время создания

**OrderDetail** — детали заказа (связующая таблица)
- `id` — идентификатор
- `order_id` — ссылка на заказ
- `dish_id` — ссылка на блюдо
- `dish_name` — название блюда на момент заказа (для истории)
- `dish_price` — цена блюда на момент заказа
- `quantity` — количество порций

### Интерфейс

Приложение состоит из трёх основных страниц:

**Главная страница (Касса):**
- Левая колонка: меню с категориями и блюдами
- Правая колонка: корзина с возможностью редактирования и форма для отправки заказа
- Блок «В работе у зала» со списком всех заказов и их статусами

**Экран повара (/kitchen):**
- Крупные карточки заказов с таймером ожидания
- Цветовая индикация: белый (до 15 мин), красный (более 15 мин)
- Кнопки для изменения статуса заказа

**Страница статистики (/stats):**
- Карточки с основными метриками
- Список популярных блюд с прогресс-барами
- Визуализация выручки по дням

### Инициализация базы данных

При первом запуске автоматически создаются таблицы. Скрипт `create_db.py` также заполняет базу тестовыми данными:
- **Категории:** Горячие блюда, Супы, Салаты, Напитки, Десерты, Закуски
- **Блюда:** 25 популярных позиций (стейки, паста, супы, салаты, напитки и десерты) с реалистичными ценами

### Запуск проекта

1. Установить зависимости:
   ```bash
   pip install sqlalchemy nicegui
   ```
2. Инициализировать базу данных (опционально):
   ```bash
   python create_db.py
   ```
3. Запустить приложение:
   ```bash
   python app.py
   ```
4. Открыть браузер: http://localhost:8080

### Технологии
- **Python 3**
- **SQLAlchemy** — ORM для работы с базой данных
- **SQLite** — легковесная встроенная база данных
- **NiceGUI** — фреймворк для создания веб-интерфейса на чистом Python
